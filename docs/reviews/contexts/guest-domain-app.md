### [D11] BLOCKER `crypto.randomUUID()` in domain layer — I/O impurity

```
File:  src/contexts/guest/domain/events.ts:35,60,85,109
Quote: eventId: crypto.randomUUID(),
Rule:  D11 — domain must not use I/O, crypto, or runtime-specific APIs. IDs via IdGenerator port.
Fix:   Accept `eventId` as a constructor argument (generated by the caller) or inject an IdGenerator port. Build.ts already provides `idGen` for entity IDs — extend the same pattern to `eventId`.
```

### [D11] BLOCKER `node:assert/strict` import in domain layer

```
File:  src/contexts/guest/domain/events.ts:4
Quote: import assert from 'node:assert/strict'
Rule:  D11 — domain imports only itself + shared/domain/. Node built-ins are forbidden.
Fix:   Replace `assert()` calls with inline guards that return `never` or throw a domain error (e.g., `if (!(args.occurredAt instanceof Date)) throw guestError('invalid_session', '…')`). Alternatively, remove assertions entirely since the domain constructors already validate inputs — the event constructors are called from application use cases that control the data flow.
```

       }
